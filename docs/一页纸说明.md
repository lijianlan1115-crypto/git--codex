# 一页纸说明：基于Codex的研发协同变更分析与测试预警工作流

## 解决的问题

多人协作开发中，开发人员经常不知道自己的修改会影响什么、影响谁、是否影响同事工作区和测试环境；同事提交代码后，也很难及时判断是否影响自己的模块。本项目用 Codex 读取本地 Git 代码库和 Git diff，结合 V27 业务知识库，自动生成影响分析、协同风险和测试清单。

## 方法

系统将本地 Git 代码库作为真实代码来源，Python 负责读取 `git status / git diff / git log`，Codex 负责理解代码变化和按需求修改代码，V27 知识库负责把代码影响翻译成业务节点、字段、Agent 和测试风险，最终输出 Markdown 与 JSON 报告。

## 结果

每次需求变更或代码变更后，系统自动输出：

- 改了什么。
- 影响哪些文件、模块、节点、Agent、输入输出和配置。
- 是否影响其他开发、测试环境和正式环境。
- 风险等级。
- 应该测试什么、怎么测试。
- 通过标准和失败标准。
- 可选：Codex 实际修改代码后生成 patch。

## 效果证明

项目支持两种演示方式：

- 只分析：读取本地 Git diff，生成影响分析和测试预警。
- 分析并修改：输入需求变更，Codex 修改本地代码库，再基于实际 diff 生成验证方案。

输出文件保存在 `outputs/change_summary-latest.md`、`outputs/change_summary-latest.json` 和 `outputs/codex_patch.diff`。
