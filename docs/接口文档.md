# 接口文档

## 鉴权

所有写入和分析接口都需要 Header：

```text
Authorization: Bearer <COZE_TOKEN>
Content-Type: application/json
```

启动前设置：

```bash
export COZE_TOKEN="test-token-123"
```

## GET /health

检查服务、默认本地 Git 代码库、runner 和 V27 知识库是否可用。

## POST /analyze-change

只分析，不修改代码。

请求示例：

```json
{
  "repo": "hotel--ota-ai",
  "requirement": "分析当前修改是否影响收益策略",
  "commit": "",
  "files": [],
  "diff": ""
}
```

字段说明：

- `repo`：`repos/` 下的本地 Git 代码库名称，默认 `hotel--ota-ai`。
- `requirement`：需求说明。
- `commit`：可选，指定本地提交进行分析。
- `files`：可选，手动指定涉及文件。
- `diff`：可选，直接传入 diff；不传则读取本地 Git diff。

## POST /apply-change

允许 Codex 修改本地代码库，并生成实际 diff、patch 和测试预警。

请求示例：

```json
{
  "repo": "hotel--ota-ai",
  "requirement": "在价格策略中增加价格下限校验，并生成验证方案",
  "files": ["runtime/hotel_ota_runtime.py"]
}
```

返回重点字段：

- `codex`：Codex 执行结果。
- `impact`：V27 影响节点、传播节点、关键词。
- `risk_level`：风险等级。
- `test_cases`：测试清单。
- `outputs`：报告和 patch 文件路径。

## GET /latest-report

读取最近一次 Markdown 报告内容。
