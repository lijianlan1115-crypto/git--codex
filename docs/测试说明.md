# 测试说明

## 1. 启动服务

```bash
cd "/Users/jelly/Public/trae-file/6.24考核/6.24考核工作流"
export COZE_TOKEN="test-token-123"
uvicorn server:app --host 127.0.0.1 --port 8000
```

如果 8000 被占用，换成 8001。

## 2. 健康检查

```bash
curl http://127.0.0.1:8000/health
```

重点确认：

- `default_repo_exists: true`
- `runner_exists: true`
- `runner_executable: true`
- V27 数据数量大于 0

## 3. 只分析模式

```bash
curl -X POST http://127.0.0.1:8000/analyze-change \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer test-token-123" \
  -d '{"repo":"hotel--ota-ai","requirement":"分析当前本地修改对价格和收益策略的影响"}'
```

预期：

- 返回 `status: success`
- 生成 `outputs/change_summary-latest.md`
- 报告包含影响范围、风险等级、测试清单

## 4. 修改代码模式

```bash
python3 scripts/apply_requirement_change.py \
  --repo hotel--ota-ai \
  --requirement "在收益策略相关逻辑中补充价格下限校验，并输出验证说明"
```

预期：

- Codex 修改 `repos/hotel--ota-ai/` 中的相关文件。
- 生成 `outputs/codex_patch.diff`。
- 生成 `outputs/change_summary-latest.md`。

## 5. 不调用 Codex 的兜底演示

如果现场网络或 Codex 登录异常，可以先演示 V27 映射能力：

```bash
python3 scripts/build_change_summary.py \
  --repo hotel--ota-ai \
  --requirement "分析当前本地修改影响" \
  --no-codex
```

该模式仍会生成报告，但 Codex 分析部分显示为未调用。
