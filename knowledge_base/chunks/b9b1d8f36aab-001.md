# 统一数据契约

## 1. 目标

所有 API、数据库、RPA、人工上传和样例数据都转换成统一对象。17 个 skill 只依赖统一对象，不直接依赖某个 API 字段或数据库原始表字段。

API 参考源与业务渠道分开记录：`adapter_vendor` 表示适配器供应商或系统，`channel_source` 表示业务渠道或数据归属。订单来了属于 PMS/直连中台适配源，不等同于 OTA 渠道。

## 2. 元信息

```json
{
 "adapter_vendor": "beyondh | meituan | dindanll | xhotel | manual | database",
 "channel_source": "meituan | feizhu | douyin | ctrip | wechat | pms | manual",
 "data_source_type": "meituan_api | beyondh_api | dindanll_api | sqlite_db | mysql_db | postgres_db | rpa | manual_upload | sample_data",
 "source_capability": "read_only | write_dry_run | write_live_pending | unavailable",
 "field_quality": "confirmed | inferred | manual_required | unavailable",
 "captured_at": "2026-06-02 10:00:00",
 "data_business_date": "2026-06-02",
 "data_snapshot_time": "2026-06-02 10:00:00",
 "freshness_status": "fresh | stale | missing_date | demo_data",
 "data_age_hours": 0.5,
 "business_status": "current | demo_or_historical",
 "today_label_allowed": true
}
```

字段含义：

- `adapter_vendor`：实际适配来源。例：美团技术服务合作中心为 `meituan`，Beyondh PMS 为 `beyondh`，订单来了开放平台为 `dindanll`，数据库统一为 `database`。
- `channel_source`：业务渠道或数据归属。例：美团 OTA 为 `meituan`，订单来了 PMS 库存数据为 `pms`。
- `data_source_type`：采集方式。API 未开通时继续使用 `sqlite_db/mysql_db/postgres_db`、`sample_data`、`manual_upload` 或 `rpa`。
- `data_business_date`：业务数据所属营业日期，不等于查询时间。
- `data_snapshot_time`：数据快照、导入或采集时间。
- `freshness_status`：今天且不超过 24 小时为 `fresh`；超过 24 小时为 `stale`；缺日期为 `missing_date`；样例为 `demo_data`。
- `today_label_allowed`：只有 `fresh` 数据允许使用“今日/实时”口径；超过 72 小时必须明确历史/演示口径。

## 2.1 开发调试元数据

以下字段只用于开发复核、问题定位或 debug 模式，不属于生产飞书常规输出字段：

- `source_governance`: 多源依据说明。
- `blueprint_sources`: 使用到的功能蓝图文件。
- `confirmation_level`: `confirmed | reference_only | configurable | needs_business_confirm | unavailable`
- `conflict_resolution`: 多源冲突时采用的处理方式。

生产飞书回复不得展示 `source_governance`、`blueprint_sources` 或来源文件清单；如用户询问依据，只能给脱敏摘要并提示详细依据在工程 `requirements/` 中维护。

## 2.2 身份与权限对象

飞书、CLI、cron、manual test 都必须带入或推导身份上下文：

```json
{
 "user_role": "admin | owner | operator | frontdesk | guest",
 "auth_context": {
 "source": "feishu | cli | cron | manual_test",
 "auth_status": "authorized | unauthorized | missing_identity",
 "feishu_user_id": null,
 "feishu_open_id": null,
 "feishu_chat_id": null,
 "permissions": []
 }
}
```

角色权限：

- `admin`：全部权限，包含角色管理和安全配置。
- `owner`：除角色管理外的全部业务权限。
- `operator`：诊断、建议、dry-run、发起审批。
- `frontdesk`：只允许接收/反馈前台任务。
- `guest`：无业务权限。

飞书来源若无法解析发送人身份，必须标记为 `guest` 或 `missing_identity`，不得触发业务 skill。

## 2.3 数据库来源规则

数据库来源和 API、RPA、人工上传并列，但必须只读：

- `adapter_vendor=database`
- `data_source_type=sqlite_db | mysql_db | postgres_db`
- `source_capability=read_only`
- 字段未映射前为 `manual_required`，映射并验证后为 `confirmed`

数据库查询必须经过 runtime 白名单模板，例如 `operating_snapshot`、`price_snapshot`、`order_snapshot`、`daily_metrics`、`monthly_metrics`。skill 和模型不得直接生成自由 SQL，也不得直接解释数据库原始表字段。

MySQL 报表库采用配置驱动映射：`/etc/hotel-ota-ai/database-source.json` 负责定义 profile、表名、字段名、指标别名、房态别名和隐私脱敏。字段、表或库变化时优先修改该私有配置，不修改 17 个 skill。
