# S5 智能收益决策

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户询问“今天要不要调价”“涨多少/降多少”“收益怎么做”“是否需要开关房价策略”时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`
- `{baseDir}/../_shared/operating-policy.md`
- `{baseDir}/../_shared/channel-api-map.md`

## 输入依赖

- S1：价格底线、价格上限、单次涨降幅限制、审批规则。
- S2：统一经营快照、出租率、ADR、RevPAR、库存、订单。
- S4/S9：需求指数、节假日、活动、行情热度、流量峰谷。
- S7：竞品价格和排名，P2 可用时接入。
- S15：销售基准线。
- S16：当前进度偏差。
- 美团参考字段：挂牌价、促销价、活动后到手价、房型可售/已售、曝光、浏览、支付转化、HOS。
- 订单来了参考字段：PMS 房型价格、底价、库存和订单状态；只用于统一数据契约，不作为 OTA 渠道。

## 执行流程

1. 先生成 dry-run 建议：

```bash
python runtime/hotel_ota_runtime.py revenue-decision --hotel-id puyue
python runtime/hotel_ota_runtime.py normalize-sample --sample meituan-price
python runtime/hotel_ota_runtime.py normalize-sample --sample dindanll-price
```

2. 对比当前进度与基准线，判断提前、正常、落后或异常。
3. 先看需求指数、OTA 健康、流量转化、库存和竞对预警，再检查价格底线、价格上限、单次涨降幅限制。
4. 按房型、渠道、日期拆分建议动作。
5. 只输出建议，不直接执行；涉及调价或房量变化时交给 S6 并要求admin/owner 审批。

## 输出要求

返回收益结论、证据、安全校验、建议动作列表、审批状态。

## 安全规则

- 默认 `mode` 为 `dry_run`。
- 不得绕过 S6 直接执行任何渠道写接口。
- API 未开通时不得失败，必须基于 `sample_data/manual_upload/rpa` 产出保守建议。
- 不得给出低于底价或超过单次涨降幅限制的建议。
