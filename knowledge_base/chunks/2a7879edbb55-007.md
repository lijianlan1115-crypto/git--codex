## 六、自动调整规则引擎

### 6.1 决策树1：动态定价调整（增强版）

```
触发：每小时评估一次
├─ 当前处于什么市场周期？
│ ├─ 淡季
│ │ ├─ 出租率<65% → 降价（直接降价>活动降价），充量优先（#1,#8）
│ │ ├─ 出租率65-80% → 价格相对稳定，不大涨不大降（#8）
│ │ └─ 出租率>80% → 小幅多频次提价，确保涨价后仍有订单（#8）
│ ├─ 平季
│ │ ├─ 出租率<70% → 稳价或小幅降价跑量（#8）
│ │ ├─ 出租率70-85% → 观察预订增速，决定是否小幅度提价（#8）
│ │ └─ 出租率>85% → 小幅多频次提价（#8）
│ └─ 旺季
│ ├─ 出租率<75% → 稳价跑量，确认流量是否充足（#8）
│ ├─ 出租率75-90% → 涨价条件检查
│ │ ├─ 同类房型库存剩余少 + 有效销售时间长 → 直接涨价（#8）
│ │ └─ 库存充足 → 稳价继续跑量
│ └─ 出租率>90% → 果断涨价，注意不同房型差异化
│ ├─ 哪个房型具备涨价条件（库存少+时间长）→ 谁先涨（#8）
│ └─ 其他房型先不变
├─ 周内周末差异化定价（#6，v4新增）
│ ├─ 周内出租率显著低于周末 → 周内降价跑量，周末稳价保收益
│ ├─ "运营做减法"：周内减低效活动
│ └─ "渠道做加法"：周内加抖音等新渠道获客（#6）
├─ 竞对大面积降价？
│ ├─ 是 → 检查商圈曝光均值
│ │ ├─ 均值低 → 需求不足，赶紧降价提排名（#8）
│ │ └─ 均值正常 → 供大于求，果断降价或稳价（#8）
│ └─ 否 → 按自身条件决策
├─ 降价方式选择
│ ├─ 库存多+销售压力大
│ │ ├─ 淡季 → 直接降价（主力房型同幅+原活动逻辑不变）（#8）
│ │ └─ 旺季 → 变相降价（限时特惠/今夜特价果断低于天天特价）（#8,#9）
│ └─ 库存适中 → 活动调价（今夜特价/限时特惠）
├─ 涨价幅度控制
│ ├─ 平季/淡季 → 小幅度多频次（+5~15元/次），确保涨价后仍有订单（#8）
│ └─ 旺季/民宿/景区 → 可大幅涨（#8）
├─ 利润派量价模型校验（#16，v4新增）
│ ├─ 计算当前调整方案在不同出租率×均价组合下的毛利
│ ├─ 若当前调整方案偏离利润最优组合>15% → 标记"收益效率偏低"预警
│ ├─ 核心判断：均价高+出租率低→利润增长（利润派）；均价低+出租率高→利润下降
│ └─ 根据酒店所属派系（满房/RevPAR/利润）校验调整方向
├─ 投资人五维策略框架校验（#21，v4新增）
│ ├─ 客群维度：当前调价是否匹配目标客群？
│ ├─ 产品维度：当前房型配置是否匹配价格策略？
│ ├─ 价格维度：调价后价格秩序是否仍然合规？
│ ├─ 渠道维度：调价是否影响各平台价格一致性？
│ ├─ 周期维度：当前是否处于正确的淡旺季策略（淡季做量/旺季保价）？
│ └─ 投资回报周期影响：回报周期长→降价空间大；回报周期短→保价优先
├─ 客源结构联动校验（v5新增，吸纳超级店长）
│ ├─ 商务客占比>60% → 周内调价空间小，偏稳价
│ ├─ 旅游客占比>60% → 节假日提前涨价空间大
│ └─ 价格敏感客占比>50% → 降价跑量效果优于涨价保收益
├─ 活动诊断联动（v5新增，吸纳超级店长）
│ ├─ 检查当前所有平台活动参与状态
│ ├─ 未参加的高效活动 → 建议参加
│ ├─ 已参加但效果差的活动 → 建议退出
│ └─ 活动折扣冲突 → 自动告警+修正建议
├─ 推广自动化决策（v5新增，吸纳甩手掌柜）
│ ├─ 流量高峰 → 自动建议开启推广
│ ├─ 流量低谷 → 自动建议关闭推广
│ ├─ 推广出价 → 根据竞对出价和转化率自动建议调出价
│ ├─ 防追价 → 监测平台间追价行为，自动建议恢复
│ └─ 推广余额为零 → 自动告警推送
└─ 安全校验
 ├─ 价格下限保护：月运营成本÷30÷房间数（#4）
 ├─ 房型价差检查：调价后房型价差不超过50元
 └─ 防追价保护：调价后检查平台间追价行为（v5新增）
```

### 6.2 决策树2：活动参与决策

```
触发：平台活动推荐/日常评估
├─ 常规活动（天天特价/新客/早定/连住）
│ ├─ 折扣梯度检查：早定<限时≈今夜<新客≤天天特价（#10）
│ ├─ 新老客均衡度检查
│ │ ├─ 常规商圈 → 新老客折扣差异不要太大（#9）
│ │ ├─ 旅游商圈 → 新客天然多，不建议单独加大新客折扣（#9）
│ │ └─ 过境商圈 → 新客低折扣抢单效果好（#9）
│ └─ 早定/连住折扣必须同时低于天天特价和新客折扣（#9,#10）
├─ 红包/补贴活动
│ ├─ 只做一个平台 → 积极参加（#11）
│ ├─ 多平台运营
│ │ ├─ 被迫参加 → 监控浏览转化率/曝光转化率
│ │ │ ├─ 转化率下降 → 果断取消（#11）
│ │ │ └─ 转化率正常或上升 → 坚持（#11）
│ │ └─ 主动选择 → 淡季谨慎（影响两级转化率），旺季可参与（#11）
│ └─ 修正方法：转化率异常→果断下掉叠加活动；非自主活动找业务经理（#11）
├─ 限时特惠/今夜特价
│ ├─ 临时降价时使用
│ │ ├─ 折扣果断低于天天特价和新客折扣（#9）
│ │ └─ 标签露出权重高，旺季效果更好（#9）
│ └─ 非常规时使用，平时尽量不用（#10）
└─ 活动诊断（v5新增，吸纳超级店长）
 ├─ 列出各平台所有可用活动
 ├─ 标注参与状态（已参加/未参加/不适合）
 ├─ 效果评估（参加后转化率变化/订单增量/ROI）
 └─ 改善建议（建议参加/建议退出/建议调整折扣）
```

## 七、价格预埋规则

1. **月度价格按三组价设计**（#8, #10）：周内原价、周末原价、节假日原价分别设定
2. **懒人法则**（#10）：预埋时只设划线价，活动折扣保持不变
3. **折扣梯度在预埋时统一校验**（#9, #10）：确保早定<限时≈今夜<新客≤天天特价
4. **平台间价格一致性**（#1, #11）：预埋后检测各平台实际展示价差≤5元
5. **价格下限保护**（#4）：所有预埋价格不得低于单房运营成本
6. **预埋后需检查平台自动追加活动**（#11, #12）
7. **预埋需结合利润测算排表**（#16, #21）：三组价的目标均价需接近利润最优组合
8. **6个月远期预埋**（v5吸纳甩手掌柜）：结合周边活动日历（演唱会/展会/赛事），提前布局远期房态和价格；远期预埋需同样经过折扣梯度校验和利润测算校验
9. **多平台划线价分别管理**（v5吸纳甩手掌柜）：美团/携程/飞猪/抖音各平台划线价分别管理，预埋时各平台独立写入

## 八、好评回复规则

1. 好评需在24小时内回复，超时未回复的标记为"待处理"
2. 软广内容自然植入，不生硬推销
3. 软广素材根据当前运营重点动态更新
4. 不同平台回复风格可差异化（携程偏正式、美团偏亲切）
5. 差评预警推送（不在本需求范围内，但需支持）
6. **精准锁定法则**（#18）：锁定5-10间品质房→看人下菜（沟通超5条锁客）→员工匹配→防差评优先
7. **好评四板斧**（#18）：出发点→交互方式→实现方式→运行管理
8. **评论缓冲期机制**（v5吸纳甩手掌柜）：AI生成回复后3小时内可修改，超时自动发布；推送到企业微信供人工审核/修改
9. **点评规则库约束**（v5吸纳超级店长）：AI回复生成时参考各平台点评规则，避免违反规则导致处罚；规则库包括评价时间窗口、删除修改规则、申诉规则、惩罚机制
