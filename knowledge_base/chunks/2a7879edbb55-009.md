#### SK06 执行引擎

| 项目 | 定义 |
|------|------|
| **编号** | SK06 |
| **名称** | 执行引擎 |
| **所属智能体** | 执行智能体 |
| **核心职责** | 统一接收所有决策Skill的执行指令，调用各平台API/RPA完成操作；支持房价同步、活动调整、推广开关、内容修改、远期价格批量写入、评论回复、前台指令推送；v5新增：微信/移动端指令执行+推广出价调整+防追价恢复+多平台划线价分别管理+6个月远期预埋执行+周边活动日历写入 |
| **输入** | ①定价指令（来自SK05）②推广指令（来自SK07）③远期预埋指令（来自SK08）④口碑回复指令（来自SK09）⑤前台操作指令（来自SK12）⑥人工确认结果 ⑦微信/移动端指令解析结果（v5新增） |
| **输出** | ①执行结果确认（成功/失败/部分成功+失败原因）②执行日志 ③预埋效果校验结果 ④前台指令送达确认 ⑤推广出价调整结果（v5新增）⑥防追价恢复结果（v5新增） |
| **优先级** | **P1** |
| **v5竞品吸纳** | 吸纳甩手掌柜：微信/移动端指令执行通道+推广出价RPA调整+防追价恢复执行+多平台划线价分别写入+6个月远期预埋执行；分级确认机制（小调整自动/大调整需确认）是区别于甩手掌柜95%全自动的核心差异化 |

**人工确认机制**：

| 操作类型 | 自动/需确认 | 说明 |
|----------|------------|------|
| 小幅调价（≤15元） | 自动 | 标准策略 |
| 折扣梯度修正 | 自动 | 按标准梯度自动修正 |
| 限时特惠/今夜特价开启 | 需确认 | 需确认折扣幅度和房间数 |
| 大幅调价（>15元） | 需确认 | 可能影响价格秩序 |
| 红包/补贴参加或取消 | 需确认 | 影响多平台平衡 |
| 关房操作 | 需确认 | 风险高 |
| 付费流量投放 | 需确认 | 涉及预算支出 |
| 推广出价自动调整 | 需确认（首次）/自动（后续） | v5新增 |
| 防追价恢复 | 自动 | v5新增，保护性操作 |
