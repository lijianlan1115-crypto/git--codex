酒店 OTA 五表数据字典
适用于真实数据表与“_模拟”测试数据表 | 版本日期：2026-06-09
本文档定义经营快照、价格、需求行情、运营诊断和销售基准线五张业务表的字段、类型、格式与计算口径。所有模拟数据仅用于功能验证和演示，不得创建正式审批或进入 live 执行。
统一类型与录入规范
项目
规范
日期
使用 YYYY-MM-DD；business_date 表示业务归属日期，不等于查询时间。
百分比
底层保存0–1的小数，例如0.8065；Excel可显示为80.65%。
金额
单位为人民币元，通常使用 decimal(10,2)。
整数
房量、订单、排名、曝光和浏览使用非负整数；健康等级按字段定义取值。
JSON string
在Excel单元格中保存合法JSON文本；数组使用[]，对象使用{}。
缺失值
未知数据使用空单元格，不以0代替；0只表示业务上确认的零值。
数据隔离
文件名含“_模拟”或位于模拟数据目录的记录一律视为 demo_data。
1. 经营快照表（operating_snapshot）
文件：

用途：
记录酒店在某个营业日的核心经营结果，为 S2 经营房态、S5 收益建议和 S16 进度偏差提供基础数据。
数据粒度：
每个酒店、每个营业日期一行。
建议唯一键：
hotel_name + business_date
关键口径：
occupancy_rate = sold_rooms / available_rooms；remaining_rooms = available_rooms - sold_rooms；revpar 通常等于 adr × occupancy_rate。

这些指标主要来自 OTA 商家后台，不应使用 PMS 的经营指标代替。无真实来源时仅可使用明确标识的模拟数据。
