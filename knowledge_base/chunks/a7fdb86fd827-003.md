## HTML 报告

报告文件名固定：

```text

```

报告样式来自：

```text
templates/ota_diagnosis_report_demo.template.html
```

每次执行都会覆盖输出目录里的旧报告，避免飞书返回旧内容。

## 飞书接入边界

触发词不写在飞书入口代码里，统一写在：

```text
config/triggers.yaml
```

飞书入口或 OpenClaw Agent 读取 `config/triggers.yaml` 后判断是否调用 S14，例如：

```python
if any(k in text for k in ["运营诊断", "S14诊断", "OTA全面诊断", "生成诊断报告"]):
 result = S14OperationDiagnosis(config).execute(inputs)
 send_text(open_id, result["feishu_message"])
```

上面代码只是说明流程；实际触发词清单必须以 `config/triggers.yaml` 为准。

飞书回复格式由 Skill 输出的 `feishu_message` 严格控制。飞书 Bot 不应自己拼接综合得分、风险等级、字段缺口等业务内容，避免 Bot 和 Skill 计算结果不一致。

推荐使用“Agent JSON -> Python 固定排版”方式：

```text
Agent 先输出 JSON
↓
Python 检查是不是合法 JSON
↓
如果是 JSON：runtime/reply_formatter.py 转成固定飞书文本
↓
如果不是 JSON：返回“诊断结果格式异常，请重新生成。”
```

Agent 输出 JSON 时，不得输出 Markdown、表格、解释文字或自然语言总结。Python 只接受 JSON 对象，最少需要 `hotel_name`、`period_start`、`period_end`、`final_score`、`report_url` 五个字段。

`feishu_message` 固定格式：

```text
【S14 酒店 OTA 诊断报告已生成】

酒店：...
周期：... 至 ...
综合得分：... / 100
风险等级：...

报告链接：
...

说明：当前为 S14 测试机器人返回结果，不影响正式酒店 OTA Agent。
```

飞书回复不得展示：

- MySQL DSN
- 原始订单明细
- 角色表
- API 密钥
- 完整 runtime JSON
- 源码和服务器内部路径
