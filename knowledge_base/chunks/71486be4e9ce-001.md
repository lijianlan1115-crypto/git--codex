# S15 销售基准线引擎

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户询问今日目标、销售基准线、小时进度线、日报基准，或其他节点需要 `target_room_nights`、`hourly_target_curve` 时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`

## 核心职责

- 生成 `sales_baseline`、`target_room_nights`、`hourly_target_curve` 和 `baseline_confidence`。
- 使用 N005 经营快照、N006 需求指数/日历上下文、历史同期数据和房量修正目标。
- 输出 `historical_same_weekday_curve`，供 S9 流量峰谷和后续偏差分析使用。
- 基准线只作为分析依据，不直接触发调价或写动作。
- 基准线计算属于半脚本化逻辑：runtime 先输出结构化曲线，skill 再解释假设、数据来源和缺失降级。

## 执行流程

1. 确认 `hotel_id` 和 `business_date`。
2. 收集 `room_total`、`operating_snapshot`、`historical_same_period`、`demand_index`、`calendar_context` 和 `demand_reasons`。
3. 优先使用历史同期/同星期曲线；历史不足时采用降级基准线，并降低 `baseline_confidence`。
4. 用需求指数、日历类型和当前经营状态修正目标间夜与小时曲线。
5. 保留兼容字段 `target_orders`、`hourly_curve`，但最终说明优先使用 V26 字段。
6. 执行：

```bash
python runtime/hotel_ota_runtime.py baseline --hotel-id puyue
```

7. 说明置信度、缺失数据和是否可作为 S16/S5/S9/S3 的下游依据。

## 输出要求

返回 `sales_baseline`、`target_room_nights`、`hourly_target_curve`、`baseline_confidence`、`historical_same_weekday_curve`、关键假设和下一次刷新时间。

## 安全规则

- 基准线不等于调价指令。
- 数据不足时必须标注“历史不足/降级基准线”，并说明置信度。
