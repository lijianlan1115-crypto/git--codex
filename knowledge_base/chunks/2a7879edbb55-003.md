**价格调整规则**：

1. **涨价条件**（#8）：同类房型库存剩余少 + 有效销售时间长 → 可涨；淡季/平季小幅度多频次（5-15元/次），旺季可大幅涨
2. **降价方式选择**（#8, #9）：
 - 库存多+淡季 → 直接降价（主力房型同幅+原活动逻辑不变）
 - 库存多+旺季 → 变相降价（限时特惠/今夜特价果断低于天天特价和新客折扣）
 - 库存适中 → 活动调价（限时/今夜特价标签露出）
3. **竞对大面积降价应对**（#8）：商圈均值低→降价提排名；商圈均值正常→供大于求，果断降价或稳价
4. **价格秩序**（#2, #8）：白低晚高或持平，白天跑量稳收益，晚上酌情提价
5. **懒人法则**（#10）：降价/涨价只调划线价，活动折扣不变，避免折扣体系混乱
6. **价格下限保护**（#4）：月运营成本成本÷30÷房间数=单房运营成本，实际价不得低于价格下限
7. **周内周末差异化定价**（#6）：周内周末出租率差异大时，执行一周三组价策略
8. **房型价差管理**（#3）：高价房好卖低价房不好卖时的多策略应对

**活动调整规则**：

9. **默认参加平台所有活动**，后期无需调整活动参与决策
10. **折扣梯度校验必须保留**（#9, #10）：早定/连住 < 限时特惠 ≈ 今夜特价 < 新客 ≤ 天天特价
11. **活动冲突检测必须保留**：检测平台自动叠加活动导致的价格异常，发现后告警
12. **限时特惠/今夜特价是临时调价工具，不是"活动"**：仅在需要临时降价时开启，达到基准线后关闭
13. **红包/补贴决策**（#11）：被迫参加→监控转化率→降则撤/常则留
14. **活动诊断**（v5吸纳超级店长）：列出平台所有活动+参与状态+效果评估+改善建议；超级店长仅列出10种活动参与状态，睿寓智营增加自动效果评估和改善建议

**推广调整规则（v5吸纳甩手掌柜推广自动化）**：

15. **先转化后流量**（#5）：支付转化率高→提流量；支付转化率低→先修正价差再推流
16. **推流画像矫正**（#14）：新开启推流后7天内转化率可能阶段性降低属正常，保持价格稳定7天以上等算法矫正
17. **推广自动化**（v5吸纳甩手掌柜）：流量峰谷识别→自动开关推广→自动调出价→防追价恢复；甩手掌柜通过RPA实现全自动，睿寓智营改为"自动建议+需确认执行"，保持控制感
18. **推广余额预警**（v5吸纳甩手掌柜）：推广账户余额为零时自动告警，推送至企业微信

### 需求5：月度价格梳理与提前预埋

**需求描述**：每月梳理一次当月价格，并且提前预埋

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F5.1 当月价格梳理 | 根据淡旺季周期、节日日历、竞对远期价格，梳理当月每天的价格策略 |
| F5.2 三组价日历生成 | 生成周内/周末/节假日三组价格 |
| F5.3 价格预埋执行 | 将价格策略提前写入各平台后台 |
| F5.4 预埋效果校验 | 预埋后检查各平台展示价是否一致 |
| F5.5 6个月远期预埋（v5新增） | 抓取周边活动（演唱会等）提前布局房态和价格，支持6个月远期预埋 |

**业务规则**：

1. **月度价格按三组价设计**（#8, #10）：周内原价、周末原价、节假日原价分别设定
2. **懒人法则**（#10）：预埋时只设划线价，活动折扣保持不变，日常调价也只动划线价
3. **折扣梯度在预埋时统一校验**（#9, #10）：确保早定<限时≈今夜<新客≤天天特价
4. **平台间价格一致性**（#1, #11）：预埋后检测各平台实际展示价差≤5元
5. **价格下限保护**（#4）：所有预埋价格不得低于单房运营成本
6. **预埋后需检查平台自动追加活动**（#11, #12）：平台可能自动叠加活动导致展示价异常
7. **预埋需结合利润测算排表**（#16, #21）：三组价的目标均价需接近利润最优组合，而非仅参考竞对
8. **6个月远期预埋**（v5吸纳甩手掌柜）：抓取未来6个月的节假日+本地大型活动（演唱会/展会/赛事），提前布局房态和价格；甩手掌柜通过节假日日历+RPA同步实现，睿寓智营增加利润测算校验

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 淡旺季周期日历 | 手动+算法（M-03） | 额外5项⑤ |
| 节日效应日历 | 手动+算法（M-04） | 201字段 |
| 竞对远期价格（3-21天） | 插件/RPA（M-07） | 额外5项② |
| 商圈属性 | 手动录入（M-01） | 额外5项③ |
| 月运营成本 | 财务（I-12） | 201字段 |
| 单房变动成本 | 财务（I-13） | 201字段 |
| 房间总数 | PMS（I-30） | 201字段 |
| 各活动折扣当前值 | OTA后台（O-29） | 额外5项① |
| 历史价格数据 | 系统存储 | 运行积累 |
| 周边活动日历（v5新增） | 系统抓取+手动 | 演唱会/展会/赛事等大型活动 |

**输出内容**：

- 当月价格日历
