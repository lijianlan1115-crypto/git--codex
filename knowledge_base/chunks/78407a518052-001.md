# 飞书输出规范

本文定义所有酒店 OTA skill 在飞书端的最终回复标准。runtime 可以输出结构化 JSON，但飞书生产回复必须由 S3 消息中台或等价模板转成中文业务摘要。

## 1. 默认回复结构

普通业务回复统一使用 6 段式：

```text
结论：一句话说结果。
数据：业务日期、快照时间、新鲜度、来源。
证据：最多 3 个关键指标或事实。
风险：数据风险、经营风险或执行风险。
建议：下一步动作。
审批：无需审批 / 仅 dry-run / 待 admin/owner 审批 / 已阻断。
```

如果缺少真实今日数据，必须把“数据”段提前说明，不得用“今日实时快报”包装。

## 2. 美观和可读性

- 先讲结论，少讲过程。
- 一条普通飞书回复控制在 6-10 行。
- 每行只表达一个意思，避免大段解释。
- 不直接贴完整 JSON、SQL、代码、字段映射、源码、配置。
- 不在群聊里贴长表格；复杂内容只发摘要，详细内容进入受控报表或 artifact。
- 指标保留必要精度：百分比通常 1-2 位小数，金额按业务需要保留 0-2 位。
- 中文业务表达优先，内部字段名只在本地 CLI、开发调试或必要证据中出现。
- 不使用“已执行”“已修改”“已回滚”等容易误导的表达，除非 runtime 或运维日志有可验证证据。

## 3. 角色差异

| 角色 | 重点 | 不展示 |
| --- | --- | --- |
| `admin` | 安全状态、审批、配置缺口、风险摘要 | 密钥、真实 ID、DSN、源码全文 |
| `owner` | 经营结论、收益风险、是否需要确认 | 内部字段、调试过程、原始数据 |
| `operator` | 诊断证据、建议动作、dry-run、待补数据 | 角色表、配置、密钥、源码 |
| `frontdesk` | 具体任务、平台、截图/反馈要求 | 收益算法、审批细节、客户隐私 |
| `guest` | 无权限提示 | 任何经营数据、价格建议、审批信息 |

## 4. 场景模板

### 经营快报

```text
结论：当前只能生成经营摘要/今日实时经营快报/历史复盘。
数据：营业日期 ...，快照 ...，新鲜度 ...，来源 ...。
证据：出租率 ...，ADR ...，RevPAR ...。
风险：...
建议：...
审批：当前不是调价审批。
```

### 进度偏差

```text
结论：当前进度正常/偏慢/偏快。
数据：实际值来源 ...，目标值来源 ...，新鲜度 ...。
证据：实际订单 ...，目标订单 ...，完成率 ...。
风险：...
建议：...
审批：如需调价，先进入 S5 dry-run 和审批。
```

### 调价建议

```text
结论：建议观察/建议 dry-run 调价/当前阻断。
数据：业务日期 ...，价格来源 ...，新鲜度 ...。
证据：出租率 ...，库存 ...，行情 ...，价格守卫 ...。
风险：...
建议：...
审批：必须 admin/owner 审批；无 `approval_id` 不执行。
```

### 订单分析

```text
结论：只输出聚合订单分析。
数据：日期 ...，来源 ...，新鲜度 ...。
证据：唯一订单数 ...，渠道占比 ...，房型占比 ...，ADR ...。
风险：不展示行级订单明细。
建议：...
审批：无执行动作。
```

## 5. 拒绝和阻断

命中以下内容必须拒绝，不再继续贴正文：

- 配置、env、角色表、数据库映射、OpenClaw 主配置。
- 源码、`SKILL.md`、`references/`、`runtime_commands.md` 全文。
- CSV/XLSX/JSON 原始数据、完整 runtime JSON、数据库原始表。
- 订单行级明细、客户姓名、房号、订单号。
- 下载/安装/部署模型、插件、应用、二进制工具。
- git/stash/reset/回滚/重启服务/修改服务器文件声明。
- 绕过审批、绕过新鲜度、用聊天手动数据创建正式审批。
- 模型 provider billing/quota/key 异常被误说成 OTA/PMS/API 余额不足。

拒绝回复也要短，格式为：

```text
结论：已阻断，原因是 ...
允许：可以输出脱敏摘要/诊断结论/证据日期/风险/建议/审批状态。
建议：...
```

## 6. 与 S3 的关系

- 各 skill 只负责产出结构化结果和业务判断。
- S3 负责把结构化结果改写成飞书最终中文回复。
- 生产优先使用 `skills/hotel-ota/s03-message-hub/templates/production/`。
- 开发调试可使用 development 模板，但仍不得输出密钥、源码、订单明细或完整 runtime JSON。
- 最终飞书回复必须通过 `feishu-output-gate` 或等价安全策略。
