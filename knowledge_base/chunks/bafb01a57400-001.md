# S15 销售基准线引擎

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户询问今日目标、销售基准线、小时进度线、日报基准，或 S16/S5/S9 需要目标进度依据时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`

## V27 架构绑定

- 本 skill 的节点、Agent、上下游和禁止动作以 `references/v27_alignment.json` 为准。
- 架构事实源只引用 `architecture/node_registry.json`、`architecture/edge_registry.json`、`architecture/scenario_chain_registry.json`。
- 字段事实源只引用 `contracts/field_registry.yaml` 和 `contracts/node_io_contract.yaml`，不要复制全量字段池。
- Demo 输入使用 `examples/demo_data/nodes/N010.json`。
- Demo 输出必须保留 `data_source_type=demo_data`，并标注 `preview_only` 或 `dry_run`。
- 若上述绑定文件缺失，必须输出 `config_gap`，不得把缺失绑定伪装成 confirmed。

## 核心职责

- 调用 runtime `baseline` 生成当前代码实际支持的 `target_orders`、`target_occupancy_rate`、`hourly_curve` 和 `progress_checkpoints`。
- 将当前 runtime 字段同步为下游接受字段：`target_orders` 等价输出为 `target_room_nights`，`hourly_curve` 等价输出为 `hourly_target_curve`。
- 结合经营快照、需求指数、业务日历和历史同期口径解释基准线假设，但不在 skill 中绕过 runtime 重新写算法。
- 输出 `baseline_confidence`、数据新鲜度、目标来源和是否可供 S16/S5/S9 下游使用。
- 基准线只作为分析依据和下游判断基准，不直接触发调价、房量、推广或其他写动作。

## 执行流程

1. 确认 `hotel_id`、营业日期和用户权限。
2. 读取本目录 `references/` 五件套和共享契约。
3. 优先使用 runtime `baseline` 输出；不要在 skill 中直接写 SQL 或解释原始数据库表字段。
4. 如果 runtime 返回 `database_evidence.sales_baseline`，优先解释其 `target_orders`、`hourly_curve`、`historical_same_weekday` 和 `historical_same_date_type`。
5. 如果缺少销售基准线，允许按 runtime 现有降级逻辑解释 `daily_metrics.room_nights` 和默认小时曲线，但必须降低置信度。
6. 执行：

```bash
python runtime/hotel_ota_runtime.py baseline --hotel-id puyue
```

7. 用当前 runtime 字段生成下游接受字段：`target_room_nights=target_orders`、`hourly_target_curve=hourly_curve`。
8. 说明 `baseline_confidence`、缺失数据、数据新鲜度，以及是否可作为 S16 当前进度判断依据。

## 输出要求

返回 `daily_target`、每日目标、目标出租率、小时目标曲线、12/16/20/23 节点目标、关键假设、数据新鲜度、基准线置信度、下游使用状态和下一次刷新时间。结构化字段同时保留当前 runtime 字段和下游接受字段。

## 安全规则

- 基准线不等于调价指令。
- 数据不足时必须标注“历史不足/降级基准线/演示估算”，并说明不能直接进入正式调价链路。
