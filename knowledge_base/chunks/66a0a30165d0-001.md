# 价格、进度、转化字段规则

本文是工程内正式规则，不记录会议过程，只沉淀当前已确认的产品口径。

## 1. 价格模型

| 字段 | 中文含义 | 使用位置 | 规则 |
| --- | --- | --- | --- |
| `ota_base_price` | OTA 后台门市价 | S5/S6 | S5 建议、S6 dry-run 的核心价格目标 |
| `ota_activity_discount_factors` | 活动折扣系数列表 | S5/S6 | 支持多层活动叠加，例如 `[0.9, 0.95]` |
| `ota_activity_discount_factor` | 综合活动折扣系数 | S5/S6 | 所有活动折扣相乘后的结果 |
| `ota_estimated_final_price` | 外网预估到手价 | S5/S6/飞书摘要 | `ota_base_price * ota_activity_discount_factor` |
| `pms_price_reference_only` | PMS 价参考值 | S5/S6 | 只作营收统计和对照，不作为 OTA 写接口目标 |
| `pms_price_used_for_execution` | PMS 价是否用于执行 | S6 | 默认必须为 `false` |
| `price_target_type` | 价格执行目标类型 | S5/S6 | 当前固定为 `ota_backend_base_price` |

执行规则：

- 当前阶段调 OTA 后台门市价，不调 PMS 价。
- 外网价由 OTA 后台价叠加平台活动、店铺活动和其他折扣后得到。
- 飞书回复必须说清“建议调的是后台门市价”，不能把后台价、PMS 价和消费者到手价混用。
- S6 dry-run 必须展示后台价、活动折扣系数、预估外网价、价格底线、最高价和审批状态。
- 真实执行仍需 `approval_id`、fresh 数据、admin/owner 审批记录和 live 开关。

## 2. 分时进度模型

| 字段 | 中文含义 | 使用位置 | 规则 |
| --- | --- | --- | --- |
| `progress_checkpoints` | 分时目标节点 | S15 | 至少包含 12 点、16 点、20 点 |
| `progress_checkpoint` | 当前采用的节点 | S16 | 按当前时间落到 12/16/20 节点 |
| `checkpoint_target_orders` | 当前节点目标订单数 | S16 | 来自 S15 基准线或 runtime 比例计算 |
| `checkpoint_completion_rate` | 当前节点完成率 | S16 | `actual_orders / checkpoint_target_orders` |
| `daily_completion_rate` | 全天目标完成率 | S16 | `actual_orders / target_orders`，只作辅助 |
| `actual_source` | 实际订单来源 | S16 | 优先 `order_snapshot.business_date/checkin_time` |
| `target_source` | 目标来源 | S16 | 优先 S15 基准线或明确日目标 |

执行规则：

- S16 判断进度时优先按节点目标，不只看全天目标。
- `actual_orders` 不得使用 `daily_metrics.room_nights` 冒充实时实际进度。
- 数据上传时间不能替代业务时间；订单按入住当夜统计。
- 次日下午 14:00 前退房且未续住的房间，按当日可售房处理。
- 任一核心字段不是可信今日口径时，不得触发 S5 正式调价链路。

## 3. 流量转化与推广字段

| 字段 | 中文含义 | 使用位置 | 规则 |
| --- | --- | --- | --- |
| `exposure` | 曝光量 | S14/S9/S16/S5 | 流量判断分母 |
| `views` | 浏览量 | S14/S9/S16/S5 | 转化漏斗中间层 |
| `clicks` | 点击量 | S14/S9/S16/S5 | 点击到订单转化参考 |
| `paid_orders` | 支付订单数 | S14/S9/S16/S5 | 支付转化分子 |
| `payment_conversion_numerator` | 支付转化分子 | S14/S9 | 通常等于支付订单数 |
| `payment_conversion_denominator` | 支付转化分母 | S14/S9 | 通常等于浏览量或平台定义分母 |
| `payment_conversion_rate` | 支付转化率 | S14/S9/S16/S5 | 不单独作为调价依据 |
| `traffic_problem` | 是否流量不足 | S16/S14/S9 | 流量不足时优先补推广/曝光 |
| `conversion_problem` | 是否转化不足 | S16/S14/S5 | 转化不足时才进入价格/活动候选 |
| `promotion_bid` | 推广出价 | S8/S10/S14 | 用于推广诊断和 ROI |
| `promotion_cost` | 推广消耗 | S8/S10/S14 | ROI 成本 |
| `promotion_budget` | 推广预算 | S8/S10/S14 | 判断是否预算不足 |
| `promotion_orders` | 推广带来订单 | S8/S10/S14 | ROI 分子 |
| `promotion_revenue` | 推广带来收入 | S8/S10/S14 | ROI 分子 |

执行规则：

- 美团、携程优先；抖音来客暂缓。
- 不只保存支付转化率，必须尽量保存曝光、浏览、点击、支付订单等分子分母。
- 只有流量不足时，优先建议推广、曝光入口和活动资源，不直接降价。
- 只有转化不足时，才允许进入价格、活动叠加、内容页修复或调价候选。

## 4. 业务边界

- S4 业务日历继续作为 S15/S16/S5 的行情上下文。
- 调休上班日不能按普通周末高价处理。
- 竞品抓取是 P2，不阻塞 S5 主逻辑。
- 当前 S14 只保留内部酒店运营诊断；99 元外部获客诊断产品后续单独建机器人或网页报告，不混入当前主链路。
