# 竞对预警算法

## 1. 功能名称

固定竞对池价格预警。

## 2. 目标

监控固定竞对池的价格区间变化，输出竞对价格风险，但不直接触发调价。

用户已明确：竞对所有价格变化只预警，不得直接进入调价。

## 3. 输入数据

- 固定竞对池。
- 每个竞对的最低价。
- 每个竞对的最高价。
- 每个竞对价格更新时间。
- 本店各房型当前价。
- 本店各房型活动后到手价。
- 本店底价。
- 需求指数。
- 房型剩余可售比例。
- OTA转化状态。
- 本店价格带配置。

## 4. 核心指标

```text
竞对价格中位值 = (竞对最低价 + 竞对最高价) / 2
竞对池最低价 = min(所有竞对最低价)
竞对池最高价 = max(所有竞对最高价)
竞对池中位价 = median(所有竞对价格中位值)
本店价格竞争指数 = 本店活动后到手价 / 竞对池中位价
竞对降价幅度 = 当前竞对池中位价 / 上次竞对池中位价 - 1
```

## 5. 预警等级

```text
0级：无明显变化。
1级：轻微变化，观察。
2级：中度变化，需要关注。
3级：强变化，需要结合需求和库存分析。
4级：极端变化，需要店长确认策略。
```

## 6. 判断逻辑

### 6.1 竞对整体降价

条件：

```text
竞对池中位价下降超过阈值
OR 多数竞对最低价下降
```

输出：

- 竞对降价预警。
- 不直接建议跟降。
- 进入综合判断：需求指数、库存、转化、本店价格带。

### 6.2 竞对整体涨价

条件：

```text
竞对池中位价上升超过阈值
```

输出：

- 竞对涨价预警。
- 如果本店需求高、库存低、转化正常，可提示调价模块考虑涨价。

### 6.3 本店明显高于竞对

条件：

```text
本店价格竞争指数 > 高价阈值
```

输出：

- 本店价格偏高预警。
- 如果转化下降、库存高，建议调价模块评估降价或活动。
- 如果需求高、库存低，不一定需要降价。

### 6.4 本店明显低于竞对

条件：

```text
本店价格竞争指数 < 低价阈值
```

输出：

- 本店价格偏低预警。
- 如果需求高或库存低，建议调价模块评估涨价。
- 如果需求低且库存高，可保持跑量。

## 7. 综合判断口径

竞对降价后，必须结合：

- 当前需求指数。
- 本店库存。
- 本店转化率。
- 本店价格带。
- 本店是否接近底价。

判断示例：

```text
竞对降价 + 本店需求低 + 库存高 + 转化正常 -> 可进入降价评估
竞对降价 + 本店需求高 + 库存低 -> 不建议跟降
竞对降价 + 本店转化低 -> 先做转化诊断
竞对涨价 + 本店需求高 + 库存低 -> 可进入涨价评估
```

## 9. 可配置参数

- 竞对池。
- 竞对价格取值方式：最低价、中位价、价格区间中点。
- 竞对降价预警阈值。
- 竞对涨价预警阈值。
- 本店高于竞对阈值。
- 本店低于竞对阈值。
- 预警更新频率。

## 10. 数据列表适配校验

### 10.1 可直接使用的数据

来自 `数据列表.txt` 的可用字段：

- 同区域/同档次酒店优惠价。
- 竞品销量排名。
- 竞品活动策略。
- 同行竞争力评分。
- 同行均值：入住间夜、满房率、销售均价。
- 同行排名：曝光、订单量维度。
- 商圈份额。
- 竞争环境中的销量、评分、排名信息。

### 10.2 算法修正

用户之前定义竞对主要看价格区间，因此价格仍是必选字段。数据列表中额外存在销量、评分、排名和活动策略，这些字段应作为增强变量，不应强制依赖。

修正后的竞对预警分层：

```text
基础版：只用竞对优惠价区间，输出价格预警。
增强版：加入竞品销量排名、评分、活动策略，输出竞争强度预警。
高级版：加入商圈份额、同行均值和同行排名，输出市场份额预警。
```

竞对综合价格指数保留：

```text
本店价格竞争指数 = 本店活动后到手价 / 固定竞对池价格中位值
```

竞争强度指数可选：

```text
竞争强度指数 =
价格压力分 × 50%
+ 竞品销量排名分 × 20%
+ 竞品评分分 × 15%
+ 竞品活动强度分 × 15%
```

### 10.3 逐字稿校验依据

逐字稿第1条确认：竞对大面积降价但本店未跟进，会造成曝光好但浏览下降。

逐字稿第4条确认：竞对低价竞争要先识别体量、成本、运营管理背景，不能盲目跟降。

逐字稿第8条确认：竞对价格变化只是动态定价三维度之一，还必须结合市场需求和自身库存。

### 10.4 准确性与效率结论

该算法准确方向是“只预警，不直接调价”。可用数据比原先更丰富，但为了产品稳定，竞对增强变量应作为解释项，不作为第一版硬触发项。
