## 一、产品概述

### 1.1 产品名称与定位

| 项目 | 内容 |
|------|------|
| 产品名称 | 睿寓智营（RuiYu Smart Operations，简称 RYSO） |
| 产品定位 | 新一代酒店AI运营工具——以数据驱动决策、智能引领增长，实现酒店运营全闭环自动化 |
| 核心价值 | 数据最全（API+插件+RPA三通道）→ 执行最强（AI自动+人机协作）→ 落地最快（三周实施法）→ 交互最便（PC+微信+移动端三端覆盖） |
| 竞品差异化 | 超越甩手掌柜的"自动执行"和超级店长的"数据监控"，实现"智能诊断→基准线→偏差归因→策略决策→可控执行→效果验证"全链路闭环 |

### 1.2 目标用户

| 用户类型 | 典型画像 | 核心诉求 |
|----------|----------|----------|
| 单体酒店业主 | 30-50岁，经营1-3家酒店，缺乏专业运营团队 | 降本增效、提升收益、减少人工依赖 |
| 区域连锁酒店管理者 | 35-50岁，管理5-20家门店 | 多店统一管理、数据汇总、标准化复制 |
| 酒店职业经理人 | 25-40岁，负责酒店整体运营 | 工具提效、数据支撑、业绩突破 |

### 1.3 产品目标

1. **首次接入即产出价值**：抓取全量数据后自动生成运营健康度诊断报告，让用户当天看到问题
2. **日常运营闭环自动化**：从基准线制定→实时监控→偏差调整→效果验证，形成完整的日运营闭环
3. **月度规划系统化**：按月梳理价格、提前预埋，减少临时操作
4. **长期运营资产沉淀**：好评回复+软广积累口碑资产、抖音内容获客拓展渠道、私域会员体系沉淀客户资产
5. **移动端随时掌控**：微信指令+实时推送+移动看板，老板不用坐办公室也能运营酒店（v5吸纳甩手掌柜三端覆盖优势）

### 需求1：首次接入全量诊断

**需求描述**：首次合作先抓取一轮全部数据，生成一个诊断报告分类并且打分

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F1.1 多源数据一次性抓取 | 接入PMS、OTA后台、竞对页面、财务系统，完成首轮全量数据采集；v5增加核心圈竞对流失数据采集 |
| F1.2 7大维度27项指标评分 | 自动计算出租率健康度、定价合理性、流量获取、转化率、收益效率、口碑、渠道稳定性；v5增加图片质量评分和运营品质分析 |
| F1.3 诊断报告生成 | 输出综合评分+各维度评分+问题定位+整改优先级排序 |
| F1.4 整改建议推送 | 针对每个不及格指标给出具体整改方向和行动清单 |
| F1.5 图片质量评分（v5新增） | 采集各平台图片质量分，纳入诊断评分体系 |
| F1.6 运营品质分析（v5新增） | 从硬件基础设施+软件服务水平两个维度评估，输出改善建议 |

**业务规则**：

1. 评分标准按7大维度权重分配：出租率健康度20%、定价合理性20%、流量获取15%、转化率15%、收益效率15%、口碑8%、渠道稳定性7%
2. 每个维度设"优秀/良好/及格/不及格"四档，不及格项自动进入优先整改清单
3. 综合评分<70分标记为"需改善"，<50分标记为"高风险"
4. 报告中问题按影响面排序：影响全局的定价/流量问题优先于局部的口碑/渠道问题
5. 诊断需包含五维策略框架校验：客群→产品→价格→渠道→周期，逐维检查策略偏差（#21）
6. 诊断需判断酒店所属收益派系：满房派/RevPAR派/利润派，不同派系的"健康"标准不同（#16）
7. 收益效率维度需跑利润测算排表，判断当前量价组合是否接近利润最优（#16, #21）
8. **图片质量评分纳入口碑维度**（v5吸纳超级店长）：各房型图片质量分低于阈值时纳入整改清单，提供改善建议（首图类型、清晰度、场景化等）
9. **运营品质分析作为独立诊断模块**（v5吸纳超级店长）：从硬件基础设施（装修/设施/卫生）和软件服务水平（前台响应/客房服务/特色服务）两个维度评估，输出策略建议

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 当日/历史库存量 | PMS（I-01~I-04） | 201字段 |
| 各房型实时可售数量 | PMS（I-17） | 201字段 |
| 房型总数与分布 | PMS（I-16） | 201字段 |
| 当日/历史出租率 | PMS计算（D-01） | 201字段 |
| 当日/历史ADR | PMS计算（D-02） | 201字段 |
| 当日/历史RevPAR | PMS计算（D-03） | 201字段 |
| 曝光量/浏览量 | OTA API（O-01~O-02） | 201字段 |
| 各级转化率 | OTA API/计算（O-04~O-08） | 201字段 |
| 竞对曝光/转化率均值 | OTA后台（O-09~O-10） | 201字段 |
| 竞对引流价 | 插件/RPA（O-17） | 201字段 |
| 各活动折扣当前值 | OTA后台（O-29） | 201字段+额外5项① |
| 点评评分/条数 | OTA API（O-24~O-25） | 201字段 |
| 好评/差评产出数 | PMS/OTA（I-25~I-26） | 201字段 |
| 月运营成本 | 财务（I-12） | 201字段 |
| 单房变动成本 | 财务（I-13） | 201字段 |
| 月租金成本 | 财务（I-14） | 201字段 |
| 月人工成本 | 财务（I-15） | 201字段 |
| 投资回报周期 | 财务（I-29） | 手动录入 |
| 商圈属性 | 手动录入（M-01） | 额外5项③ |
| 淡旺季周期日历 | 手动+算法（M-03） | 额外5项⑤ |
| 各平台日均给房量/订单量 | OTA API（O-11~O-12） | 201字段 |
| 流失订单数及结构 | OTA后台（O-13~O-16） | 201字段 |
| 核心圈竞对流失数据（v5新增） | OTA后台 | 流失酒店名/流失订单数/流失金额/流失房型 |
| 酒店图片质量分（v5新增） | OTA评分数据 | 各房型图片质量评分 |

**输出内容**：

- 运营健康度综合评分（0-100分）
- 7大维度各自评分与等级
- 优先整改项TOP3~5
- 各维度详细诊断说明
- 下一步行动清单（按今日/3日内/本周分级）
- 收益派系识别结果
