# 功能蓝图落地矩阵

## 1. 总览

功能蓝图负责业务算法，不负责 API 绑定和 OpenClaw 部署。所有蓝图逻辑先进入统一数据契约，再由 runtime 计算和 skill 解释。

| 蓝图算法 | 主 skill | 辅助 skill | runtime 模块/命令 | 状态 | 验收话术 |
| --- | --- | --- | --- | --- | --- |
| 需求指数与流量峰谷 | S4、S9 | S2、S15、S16 | `decisions/demand.py` / `demand-index` | P0/P1 可演示，P2 细化小时峰谷 | 今天需求是低谷还是旺日？ |
| 调价建议 | S5 | S1、S2、S4、S7、S14、S15、S16 | `decisions/pricing.py` / `revenue-decision` | P0/P1 dry-run | 按满房派判断今天要不要调价。 |
| OTA 健康诊断 | S14 | S2、S12 | `decisions/ota_health.py` / `ota-health` | P0/P1 可演示 | OTA 健康为什么下降？ |
| 流量转化诊断 | S14、S9 | S2、S8 | `decisions/ota_health.py` / `conversion-diagnosis` | P0/P1 可演示 | 先看转化问题，不要直接降价。 |
| OTA 活动建议 | S8 | S5、S10、S11 | `decisions/promotion.py` / `promotion-plan` | P2 dry-run | 帮我规划一个美团活动，不要突破底价。 |
| 竞对预警 | S7 | S5、S8 | `decisions/competition.py` / `competition-alert` | P2 dry-run | 竞对降价了我要不要跟？ |
| 前台执行清单 | S3、S11 | S14、S16、S12 | `decisions/tasks.py` / `frontdesk-tasks` | P0/P1 可演示 | 生成前台每 2 小时执行清单。 |
| 收益最大化方案 | S5、S10 | S15、S16 | `decisions/pricing.py`、`decisions/promotion.py` | P0/P1 收益建议，P2 ROI | 这个推广 ROI 值不值得继续投？ |
| 评论分类与运营反馈 | S12、S13 | S3、S14 | `decisions/reputation.py` / `reputation-diagnosis` | P2 dry-run | 这条差评要不要升级店长？ |
