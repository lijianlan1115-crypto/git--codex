# 资料索引与引用规范

## 1. 资料职责

| 资料 | 职责 | 使用方式 |
| --- | --- | --- |
| 用户最新指令 | 当前目标、优先级、取舍 | 最高优先级 |
| `最后一次会议讨论.md` | 交付范围、业务目标、阶段节点 | 需求和验收依据 |
| `项目进展与说明.md`、`信息清单.md` | 项目背景、酒店信息、约束 | 配置和解释依据 |
| `功能逻辑蓝图/` | 需求指数、调价、OTA 健康、转化、活动、竞对、任务、收益、口碑算法 | 开发复核和算法校准 |
| `整体架构分析(1).xlsx` | 系统分层、数据流、模块关系 | 数据契约和架构映射 |
| Beyondh/美团/订单来了文档 | 接口字段、请求构造、能力边界 | adapters 和 normalize-sample |
| OpenClaw 文档 | workspace、agent、skill、cron、飞书 | 部署教程和配置 |
| 当前工程包 | 兼容入口、目录结构、已有约束 | 实现边界 |

## 2. 引用方式

来源引用只用于开发复核、验收说明和问题追踪，不写入飞书业务 Agent 的常规执行文件。需要说明依据时，可在 `requirements/` 或问题记录中使用以下格式：

- `来源依据：功能逻辑蓝图/02-调价建议算法.md`
- `来源依据：最后一次会议讨论.md`
- `来源依据：整体架构分析(1).xlsx`
- `来源依据：meituan_biz_docs`
- `来源依据：OpenClaw skills config`

## 3. 确认等级

- `confirmed`：资料一致，或用户已经确认。
- `reference_only`：API/外部文档只作参考，能力待确认。
- `configurable`：可配置阈值，不能写死。
- `needs_business_confirm`：业务冲突或阈值待确认。
- `unavailable`：当前资料或 API 不可用。

## 4. 文件责任

- skill 的规则文件只写业务判断、降级和安全规则，不写资料出处。
- skill 的 `references/input_schema.json` 写输入字段。
- skill 的 `references/output_schema.json` 写输出结构。
- skill 的 `references/runtime_commands.md` 写可调用 CLI。
- runtime 模块写稳定计算、适配、安全和日志。
- `requirements/多源依据治理.md`、`requirements/数据字段字典与来源优先级.md` 和本文负责保存来源追踪。
