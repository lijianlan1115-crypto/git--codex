# S17 客户订单分析

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户询问客户结构、订单分析、客户等级、复购、流失风险、定向优惠建议时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`
- `{baseDir}/../_shared/channel-api-map.md`

## 数据来源

- Beyondh `Order.QueryOrders`。
- Beyondh `Order.QuerySingleOrder`。
- 订单来了 `/open/pms/third/order/get`，通过 runtime 转成统一订单对象。
- 美团订单字段待账号权限确认；可先使用后台导出或人工上传。
- OTA 订单导出或用户上传截图。

## 核心职责

- 分析订单来源、房型、入住日期、提前预订天数、ADR、复购、取消。
- 在数据允许时做客户运营分层。
- 识别前台或运营需要跟进的客户任务。
- 给出定向优惠建议，但避免过度降价。

## 执行流程

1. 获取指定日期范围订单数据。
2. API 未授权时可先用样例验证统一订单对象：

```bash
python runtime/hotel_ota_runtime.py normalize-sample --sample dindanll-order
```

3. 聊天输出中脱敏个人信息。
4. 按渠道、房型、客户行为、客户价值分层。
5. 输出跟进任务和优惠建议。

## 输出要求

返回客户/订单分层、核心洞察、复购/流失风险、建议动作、数据缺口。

## 安全规则

- 不输出客人手机号、证件号、完整姓名等敏感信息。
- 不自动发送优惠或联系客人，只生成建议和任务。
