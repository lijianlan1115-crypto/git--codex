### 需求2：每日销售基准线与周汇总

**需求描述**：结合历史订单情况，生成一个每日销售基准线，诊断报告每周汇总完成情况

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F2.1 每日销售基准线计算 | 基于历史订单+淡旺季周期+竞对情况，计算每日目标出租率、目标间夜数、目标均价 |
| F2.2 基准线动态调整 | 根据市场变化（节日/竞对异动/天气）自动微调基准线 |
| F2.3 每日完成度追踪 | 实时对比实际销售进度与基准线，计算偏差率 |
| F2.4 周度汇总报告 | 汇总本周每日完成情况，分析达标/未达标原因，输出周报 |
| F2.5 实时经营指标推送（v5新增） | 每小时推送经营指标到企业微信，含本店状况+对标状况+成交状况 |

**业务规则**：

1. 基准线计算需区分淡旺季：淡季基准线侧重出租率目标，旺季侧重均价目标（#21：淡季做量，旺季保价）
2. 基准线需考虑"六八坎"：18:00前出租率应≥60%（#2），否则当晚满房概率极低
3. 周汇总中，连续3天未达基准线的项目标记为"持续偏离"，需要启动调整策略
4. 基准线不是固定值，需结合节日日历在节前/节后自动修正（#7）
5. 周汇总报告中区分"今天的事"和"远期的事"（#12：订单数≠间夜量）
6. 基准线生成需基于利润测算排表，确保基准线对应的量价组合接近利润最优（#16, #21）
7. 周汇总需评估当前策略是否符合酒店所属收益派系的目标（满房派看出租率、RevPAR派看RevPAR、利润派看毛利）（#16）
8. 投资回报周期影响基准线策略：回报周期越长→价格弹性越大→降价跑量空间大；回报周期越短→对管理/服务/价格要求越高→保价优先（#21）
9. **实时经营指标推送**（v5吸纳甩手掌柜）：每小时推送一次，内容包括：预定率/出租率/涨跌幅/行情判断/对标酒店状况/成交状况，推送至企业微信群；甩手掌柜是每小时推送到微信群，睿寓智营改为"偏差触发+定时汇总"双模式——偏差悬殊时即时推送，无偏差时按小时汇总推送，减少信息噪音

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 历史订单数据（近90天） | PMS（I-05~I-07） | 201字段 |
| 远期预订间夜数 | PMS（I-06） | 201字段 |
| 淡旺季周期日历 | 手动+算法（M-03） | 额外5项⑤ |
| 节日效应日历 | 手动+算法（M-04） | 201字段 |
| 竞对远期价格（3-21天） | 插件/RPA（M-07） | 额外5项② |
| 商圈供需指数 | OTA计算（M-10） | 201字段 |
| 月运营成本 | 财务（I-12） | 201字段 |
| 单房变动成本 | 财务（I-13） | 201字段 |
| 酒店体量（房间总数） | PMS（I-30） | 201字段 |
| 预订增速 | PMS计算（D-09） | 201字段 |
| 月租金成本 | 财务（I-14） | 201字段 |
| 月人工成本 | 财务（I-15） | 201字段 |
| 投资回报周期 | 财务（I-29） | 手动录入 |

**输出内容**：

- 每日销售基准线（目标出租率/目标间夜/目标均价，按房型拆分）
- 实时完成进度看板
- 周度汇总报告（7天达标率、偏差分析、趋势图）
- 持续偏离预警
- 利润测算排表（当前基准线对应的毛利预估）（#21）
- 实时经营指标推送（每小时汇总，含本店+对标+成交）（v5新增）

**交互方式**：基准线每日自动生成，周报每周一自动生成并推送；实时经营指标按小时推送至企业微信

**竞品对标**：

| 维度 | 甩手掌柜 | 超级店长 | 睿寓智营 |
|------|----------|----------|----------|
| 基准线 | 无"基准线"概念，有实时经营指标报告（预定率/涨跌幅/行情判断），但未明确展示目标值 | 无基准线概念，有经营业绩月度数据和出租率评级 | **独创概念**：每日销售基准线（目标出租率×目标均价×目标间夜），两个竞品都没有，是核心差异化优势 |
| 实时推送 | ✅ 每小时推送实时经营指标到企业微信群（本店状况+对标状况+成交状况） | 无实时推送功能 | **吸纳**：每小时推送经营指标，但改为"偏差触发+定时汇总"双模式，减少信息噪音 |
| 周汇总 | 未明确周度汇总功能 | 有月度经营业绩数据 | **差异化优势**：周度汇总含偏差分析+持续偏离预警+收益派系校验 |

### 需求3：每小时销售进度监控与偏差分析

**需求描述**：每小时抓取一次销售进度，如果销售进度悬殊较大分析流量和转化率数据

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F3.1 每小时销售进度抓取 | 按小时采集当前出租率、新增订单、库存变化 |
| F3.2 偏差检测 | 实际进度与基准线对比，计算偏差率 |
| F3.3 偏差悬殊时自动分析 | 偏差超过阈值时，自动钻取流量数据和转化率数据定位原因 |
| F3.4 原因诊断输出 | 判断是流量问题、转化问题还是价格问题，输出诊断结论 |
| F3.5 竞对一转二转数据对比（v5新增） | 采集竞对曝光→浏览→下单转化率数据，与自身对比定位差距 |

**业务规则**：

1. "偏差悬殊"定义：当前出租率低于基准线同期15个百分点以上，或连续2小时进度停滞
2. 偏差分析优先检查流量层级：
 - 曝光量低 → 流量获取问题
 - 曝光正常但浏览低 → 一级转化率问题（首图/标题/引流价）
 - 浏览正常但下单低 → 二级转化率问题（房型价差/关房/取消政策）
 - 下单正常但支付低 → 支付转化率问题
3. 需区分"今天的事"和"远期的事"：当天新增订单包含远期入住订单，不能简单用订单数判断当日进度（#12）
4. 18:00未达"六八坎"（出租率<60%）需触发橙色预警
5. **降价效果归因**（#7）：已降价但效果不明显时，需通过"曝光对比均值判断法"诊断
6. **订单不稳定归因**（#12）：订单忽多忽少时，执行三步排查
7. **流失分析**（#13）：流失率异常偏高时，执行流失拆分四步法
8. **转化率诊断**（#14）：转化率偏低时按三级逐层修正
9. **竞对转化漏斗对比**（v5吸纳超级店长）：当自身转化率偏低时，自动调取竞对的曝光→浏览→下单漏斗数据，定位是哪个环节被竞对拉开差距

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 当前出租率 | PMS计算（D-01） | 201字段 |
| 当日新增订单数 | PMS（I-07） | 201字段 |
| 各房型实时可售数量 | PMS（I-17） | 201字段 |
| 曝光量/浏览量 | OTA API（O-01~O-02） | 201字段 |
| 一级/二级/支付转化率 | OTA/计算（O-04~O-08） | 201字段 |
| 竞对曝光/转化率均值 | OTA后台（O-09~O-10） | 201字段 |
| 商圈曝光均值 | OTA API（O-30） | 201字段 |
| 活动冲突状态 | OTA后台（O-20, O-27） | 201字段 |
| 流失订单数 | OTA后台（O-13） | 201字段 |
| 流失按房型拆分 | OTA后台（O-16） | 201字段 |
| 竞对是否使用付费推广 | 插件/判断（O-19） | 201字段 |
| 操作日志 | OTA后台（O-21） | 201字段 |
| 竞对转化漏斗数据（v5新增） | OTA后台 | 曝光→浏览→下单各环节转化率 |

**输出内容**：

- 每小时进度快照
- 偏差预警通知（推送至企业微信/短信）
- 偏差原因诊断（流量问题/转化问题/价格问题/竞争问题）
- 降价无效归因结论（#7）
- 订单不稳定归因结论（#12）
- 流失分析报告（#13）
- 转化率诊断建议（#14）
- 建议调整方向
- 竞对转化漏斗对比报告（v5新增）

**交互方式**：自动执行，偏差悬殊时自动推送预警

**竞品对标**：

| 维度 | 甩手掌柜 | 超级店长 | 睿寓智营 |
|------|----------|----------|----------|
| 偏差检测 | 每小时自动调价+推送实时指标，有涨跌幅/行情判断，但无"基准线偏差检测" | 非每小时级别监控，有流量数据但无偏差分析逻辑 | **差异化优势**：基准线对比+偏差阈值检测+自动归因，两个竞品都没有 |
| 竞对漏斗 | 未涉及 | ✅ 同行竞对转化漏斗（曝光→浏览→下单对比），直观看到竞对差异 | **吸纳**：偏差分析时自动调取竞对漏斗数据，定位被拉开差距的具体环节 |
| 归因分析 | 有行情判断（极差/自动判断），但无系统性归因 | 有预警通知和监控通知，但无归因逻辑 | **差异化优势**：4项归因增强（降价无效/订单不稳定/流失/转化率），远超竞品 |

### 需求4：智能调整闭环（内容/价格/广告/活动）

**需求描述**：进行调整：调整内容，价格，广告，活动，达到基准线就恢复，没有达到继续调整

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F4.1 价格调整 | 根据偏差原因，执行动态定价（涨价/降价/活动调价） |
| F4.2 内容调整 | 调整房型排列顺序、首图切换（白天/晚上）、房型描述优化 |
| F4.3 广告调整 | 付费推流的开关与出价调整；v5吸纳推广自动化（流量峰谷识别→自动开关→自动调出价→防追价恢复） |
| F4.4 活动调整 | 开启/调整限时特惠、今夜特价等临时调价工具；v5增加活动诊断功能 |
| F4.5 恢复机制 | 达到基准线后自动恢复到调整前的策
