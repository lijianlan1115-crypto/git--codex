# 收益最大化方案算法

## 1. 功能名称

收益最大化方案。

## 2. 目标

根据酒店策略模式，输出当日、未来7天或月度的收益最大化建议。

该功能不是单点调价，而是综合出租率、ADR、RevPAR、库存、价格、渠道和团队执行形成策略方案。

## 3. 策略模式

### 3.1 满房派

适用：

- 中小单体酒店。
- 用户强调空置就是损失。
- 底价以上出租即盈利。
- 当前经营目标是出租率优先。

核心目标：

```text
尽量提高出租率，在满房或接近满房后再小幅提价。
```

### 3.2 RevPAR派

适用：

- 中高端酒店。
- 大体量连锁。
- 更关注均价和出租率平衡。

核心目标：

```text
RevPAR = ADR × 出租率
```

### 3.3 利润派

适用：

- 投资人视角。
- 能拿到固定成本和变动成本。
- 希望找到毛利最高的量价组合。

核心公式：

```text
毛利 = 房费收入 - 固定成本 - 变动成本
房费收入 = ADR × 已售间夜
变动成本 = 单房变动成本 × 已售间夜
```

## 4. 输入数据

- 策略模式。
- 房间总数。
- 当前出租率。
- 当前 ADR。
- 当前 RevPAR。
- 各房型库存。
- 各房型价格。
- 需求指数。
- OTA健康分。
- 竞对预警。
- 活动状态。
- 固定成本。
- 单房变动成本。
- 目标利润。
- 历史同期经营数据。

## 5. 满房派算法

### 5.1 判断逻辑

```text
1. 当前出租率是否低于目标？
 是 -> 优先跑量。

2. 库存是否集中在某些房型？
 是 -> 对滞销房型降价、活动或前台引导。

3. 18点前出租率是否低于60%？
 是 -> 当天满房压力大，进入跑量策略。

4. 是否接近满房？
 是 -> 对低库存房型稳价或小幅涨价。
```

### 5.2 输出策略

- 降价跑量。
- 缩小房型价差。
- 开启限时特惠或今夜特价。
- 前台重点推滞销房。
- 晚间库存少时小幅涨价。

## 6. RevPAR派算法

### 6.1 判断逻辑

```text
目标RevPAR = 用户输入或历史同属性RevPAR × 目标系数
RevPAR缺口 = 目标RevPAR - 当前预计RevPAR
```

如果：

```text
RevPAR低 + 出租率低 -> 优先补出租率
RevPAR低 + 出租率正常 + ADR低 -> 提价或减少低价活动
RevPAR达标 + 库存低 -> 涨价
RevPAR达标 + 库存高 -> 稳价或轻促
```

### 6.2 输出策略

- 调整目标房型价格。
- 控制低价房成交占比。
- 避免为满房牺牲过多均价。
- 在需求旺时提高 ADR。

## 7. 利润派算法

### 7.1 毛利模拟表

按不同出租率和 ADR 组合试算。

```text
已售间夜 = 可售房间数 × 出租率
房费收入 = ADR × 已售间夜
总变动成本 = 单房变动成本 × 已售间夜
毛利 = 房费收入 - 固定成本 - 总变动成本
```

系统可生成如下矩阵：

```text
出租率：60%、70%、80%、90%、95%
ADR：200、240、280、320、360
输出：每个组合对应毛利
```

选择毛利最高的组合作为建议目标。

### 7.2 输出策略

- 最优出租率区间。
- 最优 ADR 区间。
- 是否值得继续降价换出租率。
- 是否应该放弃部分低价订单保护利润。

### 11.2 算法修正

收益最大化方案可以直接支持满房派、RevPAR派和利润派三套模式。

满房派：

```text
目标 = 提高出租率和满房率
核心输入 = 满房率、预计出租率、剩余库存、需求指数、底价安全
```

RevPAR派：

```text
目标 = 提高预计RevPAR
核心输入 = 预计ADR、预计出租率、预计RevPAR、同行销售均价、历史同期RevPAR
```

利润派：

```text
目标 = 提高结算后利润
核心输入 = 房费收入、手续费、退款、结算金额、单房变动成本、固定成本
```

如果拿不到固定成本和单房变动成本，则利润派只能输出“准利润”：

```text
准利润 = 结算金额 - 手续费 - 退款金额
```

### 11.3 逐字稿校验依据

逐字稿第16条确认：收益最大化分为满房派、RevPAR派和利润派。

逐字稿第21条确认：收益高手看出租率、平均房价、单房收益和毛利四个抓手。

逐字稿第21条确认：利润派需要明确每销售一间房的变动费用，再找出利润最高的出租率和均价组合。

### 11.4 准确性与效率结论

该算法可直接进入较高可用状态，因为数据列表中已经包含 ADR、RevPAR、营收、结算、手续费和退款。唯一缺口是固定成本与单房变动成本，需要作为用户配置项。
