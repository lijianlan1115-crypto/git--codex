# 中文输出模板

飞书生产回复以 `requirements/飞书输出规范.md` 为准。除非用户明确要求其他形式，默认使用 6 段式：

```text
结论：一句话说结果。
数据：业务日期、快照时间、新鲜度、来源。
证据：最多 3 个关键指标或事实。
风险：数据风险、经营风险或执行风险。
建议：下一步动作。
审批：无需审批 / 仅 dry-run / 待 admin/owner 审批 / 已阻断。
```

涉及执行敏感动作时，必须附动作摘要：

```text
动作：price_update
酒店：...
房型：...
渠道：...
日期：...
原值：...
建议值：...
审批：需要 admin/owner 确认
```

飞书消息要求：

- 先讲结论，少讲过程。
- 普通回复控制在 6-10 行；复杂诊断先发摘要。
- 不直接贴完整 JSON、SQL、代码、字段映射、源码、配置、订单明细或原始数据。
- 数据缺失、旧数据、demo 数据必须在“数据”段说明，不得称为今日实时。
- admin 消息可包含角色、安全配置、审批和风险摘要，但不得包含真实密钥。
- 老板消息突出结果、风险、是否需要确认。
- 运营消息突出任务、证据、下一步。
- 前台消息只保留具体动作、平台、截图/反馈要求。
- guest 消息只提示“当前飞书账号未授权，请联系超级管理员加入角色表”，不附经营数据。
