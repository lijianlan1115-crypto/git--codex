# S8 推广投放

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户询问是否打开推广、是否加预算、是否参加活动、如何安排平台流量动作时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`
- `{baseDir}/../_shared/operating-policy.md`

## 核心职责

- 诊断推广余额、预算、流量缺口和转化缺口。
- 读取美团活动配置、推广余额、推广消耗等参考字段；这些字段属于 P2，不阻塞 P0/P1。
- 推荐推广开关时间窗口。
- 输出预算和出价建议。
- 涉及真实执行时交给 S11，并要求admin/owner 审批。

## 执行流程

1. 读取 S2 经营快照、S7 竞品、S9 流量峰谷、S10 ROI 和统一活动字段。
2. 判断问题是流量、转化、价格还是内容。
3. 只有在流量是主要瓶颈且安全边界明确时，才建议加推广。
4. 执行前必须进入审批链路。

## 输出要求

返回推广计划、预算区间、预期收益、风险、审批要求。

## 安全规则

- 不得直接修改推广预算、出价或活动。
- 数据不足时优先生成数据采集任务。
