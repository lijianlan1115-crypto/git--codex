# 流量转化诊断算法

## 1. 功能名称

流量/转化诊断。

## 2. 目标

当订单少、出租率低、降价无效或 OTA健康分下降时，判断问题发生在曝光、浏览、提交、支付哪个环节，并输出对应调整建议。

## 3. 输入数据

- 曝光量。
- 浏览量。
- 提交订单量。
- 支付订单量。
- 曝光浏览转化率。
- 浏览提交转化率。
- 提交支付转化率。
- 浏览支付转化率。
- 当前价格。
- 活动后到手价。
- 房型价差。
- 房型库存。
- 首图类型。
- 酒店名称关键词。
- 点评分。
- 点评量。
- 取消政策。
- 确认时长。
- 会员权益。
- 活动列表。
- 平台间价格差。
- 竞对价格指数。

## 4. 核心指标

```text
一级转化 = 浏览量 / 曝光量
二级转化 = 提交订单量 / 浏览量
三级转化 = 支付订单量 / 提交订单量
总转化 = 支付订单量 / 曝光量
```

如果没有提交订单量，可简化为：

```text
浏览支付转化率 = 支付订单量 / 浏览量
```

## 5. 课程诊断逻辑

课程中的基本判断：

- 曝光少：能触碰到的市场需求少。
- 浏览少：获取到的准流量少。
- 订单少：成交数量少。
- 先做转化，再做流量。
- 转化率未修正时，付费流量可能是在帮竞对引流。

## 6. 判断树

```text
1. 曝光是否低于历史或目标？
 是 -> 曝光问题。
 否 -> 进入浏览转化。

2. 曝光正常但浏览低？
 是 -> 一级转化问题。
 否 -> 进入支付转化。

3. 浏览正常但支付低？
 是 -> 二级/三级转化问题。
 否 -> 进入库存和价格诊断。

4. 曝光、浏览、支付都正常但订单少？
 是 -> 检查库存、房型关闭、取消率、价格带、竞对。
```

## 7. 问题分类与建议

### 7.1 曝光少

可能原因：

- 市场需求低。
- 排名下降。
- 远期房态异常。
- 房型大量关房。
- 价格缺乏竞争力。
- 平台活动缺失。

建议动作：

- 先确认需求指数。
- 检查远期3-21天房态。
- 检查是否主力房型不可售。
- 如果转化健康，可考虑加推广。
- 如果转化不健康，先修转化。

### 7.2 一级转化低：曝光到浏览低

课程调整因素：

- 位置。
- 名称关键词。
- 点评分和点评条数。
- 首图吸引力。
- 引流价格。

建议动作：

- 调整酒店名称商圈覆盖词。
- 首图匹配主客群。
- 淡季非星级酒店可优先房型图。
- 检查引流价是否过高。
- 检查是否有竞对大面积降价。

### 7.3 二级转化低：浏览到提交低

课程调整因素：

- 引流房型类别。
- 引流价及价差。
- 预订取消政策。
- 确认时长。
- 部分房型是否关房。

建议动作：

- 好房引流。
- 房型排列使用大双大双或双大双大。
- 缩小房型价差。
- 大床双床同时有房可售。
- 放宽取消政策。
- 检查确认时长。

### 7.4 三级转化低：提交到支付低

课程调整因素：

- 会员权益。
- 提前入住。
- 延迟退房。
- 积分兑换。
- 价格犹豫。
- 跨平台价格差。
- 红包补贴冲突。

建议动作：

- 检查多平台价格一致性。
- 检查是否某平台补贴导致另一平台支付转化下降。
- 增加或露出会员权益。
- 用限时特惠/今夜特价刺激支付。

## 8. 降价判断

课程线索：

- 降价不是第一手段。
- 转化率低时，先用一级、二级、特殊数据分析三套方法。
- 三天内转化率仍低于竞对均值，再考虑降价引流测试 1-3 天。

第一版规则：

```text
如果转化低且未完成转化修正动作 -> 不建议直接降价
如果转化低且已连续3天低于竞对 -> 允许降价测试
如果需求低、库存高、转化正常 -> 可降价跑量
```

```json
{

 "bottleneck": "浏览到支付",
 "main_reason": [

```

- 三级转化目标。

- 降价测试周期。

## 11. 数据列表适配校验

### 11.1 可直接使用的数据

来自 `数据列表.txt` 的可用字段：

- 平台曝光量。
- 浏览人数。
- 支付转化率。
- 点击量。
- 搜索转化率。
- 一转/二转数据。
- 市场营销标签。
- 流量峰谷时段。
- 流失订单数、流失间夜、流失金额。
- 同行排名：曝光、订单量维度。
- 同行均值。
- 确认方式。
- 入住/离店时间。
- 取消政策。
- 儿童/加床政策、宠物政策、押金要求。
- 优惠详情/标签。
- 价格说明。

### 11.2 算法修正

如果平台能提供“一转/二转数据”，优先使用平台口径，不强行用曝光、浏览、支付反推。

优先级：

```text
平台一转/二转数据 > 搜索转化率 + 支付转化率 > 曝光/浏览/支付自算漏斗
```

修正后的诊断路径：

```text
1. 搜索转化率低 -> 搜索词、名称、商圈标签、排名、首图、点评问题。
2. 一转低 -> 曝光到浏览问题，重点看价格、首图、评分、排名。
3. 二转低 -> 浏览到支付问题，重点看房型、价差、政策、确认方式。
4. 支付转化率低 -> 重点看跨平台价格、活动叠加、取消政策、价格犹豫。
5. 流失金额高 -> 进入订单流失诊断，判断价格流失还是产品流失。
```

### 11.3 逐字稿校验依据

逐字稿第5条确认：曝光量、浏览量、订单量对应不同问题，需逐级诊断。

逐字稿第5条确认：一级转化看位置、名称关键词、点评分/条数、首图、引流价格。

逐字稿第5条确认：二级转化看引流房型、引流价和价差、预订取消政策、确认时长、是否关房。

逐字稿第14条确认：降价是最后手段，先做一级和二级转化修正，连续低于竞对再降价测试。

### 11.4 准确性与效率结论

该算法需要从“自算漏斗”升级为“平台口径优先”。这样可以减少不同 OTA 平台字段定义不一致带来的误判。
