# S14 酒店运营诊断

## Reference 导航

执行本 skill 前先读取本目录 `references/` 五件套：

- `references/input_schema.json`：输入字段和字段质量。
- `references/output_schema.json`：统一输出结构。
- `references/rules.md`：多源依据、判断逻辑、异常和安全规则。
- `references/examples.md`：飞书话术、样例输入、runtime 输出和最终回复。
- `references/runtime_commands.md`：可调用的稳定脚本命令。

## 适用场景

当用户要求完整酒店诊断、OTA 诊断、内容优化、周诊断、整改任务清单时，使用本 skill。

## 必须读取

- `{baseDir}/../_shared/common-contract.md`
- `{baseDir}/../_shared/operating-policy.md`
- `{baseDir}/../_shared/channel-api-map.md`

## 核心职责

- 诊断平台健康、HOS/PSI、内容、房型展示、价格梯度、流量、转化、口碑、推广。
- OTA 健康优先读取美团 HOS、信息分、经营质量、产能水平、商家经营分和同行竞争力评分。
- 流量转化优先读取曝光、浏览、支付转化率、一转/二转、订单量和流失数据。
- API 未开通时先读取 runtime 样例归一化输出，确认字段能进入统一数据契约。
- 输出 A/B 优先级任务：A 为必须完成项，B 为持续改善项。
- 聚焦动作，不输出空泛长报告。

## 执行流程

1. 收集 S2 快照、统一数据契约字段和 OTA/PMS 截图或导出数据。
2. API 未授权时用 runtime 验证样例字段：

```bash
python runtime/hotel_ota_runtime.py normalize-sample --sample meituan-price
python runtime/hotel_ota_runtime.py normalize-sample --sample meituan-room-count
python runtime/hotel_ota_runtime.py normalize-sample --sample dindanll-inventory
```

3. 对比目标阈值、历史数据和配置竞品。
4. 把问题分为 A/B 任务。
5. 分配负责人：老板、运营、前台。
6. 通过 S3 发送摘要；长报告以 artifact 保存。

## 输出要求

返回诊断摘要、记分卡、A/B 任务、负责人、期限、预期影响。

## 安全规则

- 诊断只生成任务，不直接执行平台操作。
- 数据不足时标注缺失项和置信度。
