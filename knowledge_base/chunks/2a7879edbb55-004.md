### 需求6：好评回复与软广植入

**需求描述**：好评回复，并且在回复中增加软广内容

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F6.1 好评自动检测 | 实时检测各平台新增好评 |
| F6.2 回复内容生成 | AI生成个性化好评回复，内嵌软广内容 |
| F6.3 软广内容管理 | 软广素材库管理（酒店特色、近期活动、房型亮点等） |
| F6.4 回复执行 | 回复内容经确认后自动发布到各平台 |
| F6.5 评论缓冲期机制（v5新增） | AI回复后3小时内可修改，超时自动发布 |
| F6.6 点评规则库（v5新增） | 美团/携程/飞猪/抖音各平台点评规则详细对比，指导AI回复策略 |

**业务规则**：

1. 好评需在24小时内回复，超时未回复的标记为"待处理"
2. 软广内容自然植入，不生硬推销，格式如："感谢好评！我们的XX房型最近升级了XX，期待您再次入住体验~"
3. 软广素材根据当前运营重点动态更新（如淡季推优惠、旺季推品质）
4. 不同平台回复风格可差异化（携程偏正式、美团偏亲切）
5. 差评回复不在本需求范围内，但系统需支持差评预警推送
6. **好评精准锁定法则**（#18）
7. **好评四板斧**（#18）
8. **评论缓冲期机制**（v5吸纳甩手掌柜）：AI生成回复后推送到企业微信，3小时内可人工修改，超时自动发布；甩手掌柜的3小时缓冲期是实用设计，避免AI回复不当无法修改
9. **点评规则库**（v5吸纳超级店长）：维护各平台点评规则的详细知识库，包括评价时间窗口、删除修改规则、申诉规则、惩罚机制；AI回复时参考规则库，避免违反平台规则导致处罚；超级店长仅做规则展示，睿寓智营将规则融入AI回复生成逻辑

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 各平台点评内容 | OTA API（O-24~O-25） | 201字段 |
| 好评产出数 | PMS/OTA（I-25） | 201字段 |
| 差评产出数 | PMS/OTA（I-26） | 201字段 |
| 酒店特色/设施信息 | PMS | 201字段 |
| 当前运营重点/活动信息 | 系统配置 | 运营维护 |
| 软广素材库 | 系统配置 | 运营维护 |
| 回复风格配置 | 系统配置 | 运营维护 |
| 点评规则库（v5新增） | 平台公开规则 | 美团/携程/飞猪/抖音各平台点评规则 |

**输出内容**：

- 待回复好评列表
- AI生成的回复内容（含软广）
- 差评预警通知
- 回复执行状态
- 软广素材库
- 缓冲期内回复预览+修改入口（v5新增）
- 各平台点评规则速查（v5新增）

**交互方式**：AI自动生成回复，3小时缓冲期内可修改，超时自动发布（v5更新）；可设置"自动回复"模式（高分好评自动回复，低分好评人工审核）

**竞品对标**：

| 维度 | 甩手掌柜 | 超级店长 | 睿寓智营 |
|------|----------|----------|----------|
| AI回复 | ✅ AI自动回复，3小时内自动回复，期间可修改 | 有好评等级话术设置框架（暂无数据） | **吸纳缓冲期**：3小时缓冲期+超时自动发布+企微修改入口 |
| 软广植入 | 无软广植入功能 | 无软广植入功能 | **差异化优势**：软广素材库+动态更新+精准锁定法则 |
| 点评规则 | 无 | ✅ 美团/携程点评规则详细对比（评价时间窗口/删除修改/申诉/惩罚） | **吸纳+融入**：规则库不仅展示，更融入AI回复生成逻辑，避免违规 |

### 需求7：整改情况实时分析表

**需求描述**：整改情况的实时分析表，完成情况监控

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F7.1 整改任务清单 | 从诊断报告中提取整改项，生成可跟踪的整改任务 |
| F7.2 完成情况实时监控 | 实时追踪每项整改的执行状态和效果 |
| F7.3 整改效果量化 | 对比整改前后的指标变化，量化整改效果 |
| F7.4 整改进度看板 | 可视化展示整改整体进度 |

**业务规则**：

1. 整改任务来源：首次诊断报告 + 周汇总中发现的新问题 + 日常监控预警
2. 每项整改任务包含：问题描述、目标指标、当前值、目标值、截止时间、负责人
3. 整改效果量化：对比整改前7天与整改后7天的指标变化，计算改善幅度
4. 整改任务状态流转：待执行 → 执行中 → 待验证 → 已完成 / 未达标需继续
5. 超过截止时间未完成的整改任务自动升级预警

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 诊断报告中的问题项 | 需求1产出 | 系统内部 |
| 周汇总中的偏差项 | 需求2产出 | 系统内部 |
| 日常监控预警记录 | 需求3产出 | 系统内部 |
| 各项运营指标实时值 | PMS/OTA | 201字段 |
| 调整操作执行记录 | 需求4产出 | 系统内部 |

**输出内容**：

- 整改任务看板（待执行/执行中/待验证/已完成）
- 每项整改的进度和效果数据
- 整改完成率统计
- 超期预警

**交互方式**：自动跟踪，人工更新整改状态；看板实时展示

**竞品对标**：

| 维度 | 甩手掌柜 | 超级店长 | 睿寓智营 |
|------|----------|----------|----------|
| 整改跟踪 | 诊断报告中有"优化建议"，但无整改任务清单、完成情况监控、效果量化看板 | 有预警通知和监控通知，但无系统性整改跟踪 | **差异化优势**：整改清单→执行跟踪→效果量化→闭环验证，两个竞品都没有 |

### 需求8：订单分析与顾客累计到店数据

**需求描述**：订单分析功能，生成顾客累计到店数据

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F8.1 订单多维分析 | 按时间/房型/渠道/客源等多维度分析订单数据 |
| F8.2 顾客累计到店数据 | 统计每个顾客的累计入住次数、累计间夜、累计消费 |
| F8.3 客户价值分级 | 基于累计数据自动进行客户分级（AAA/AA/A/B/C） |
| F8.4 客源结构分析 | 分析新客/老客/会员/协议客占比；v5增强：本地/异地+休闲/商旅+价格敏感度 |
| F8.5 消费排名与VIP客户管理（v5新增） | 消费排名前20客户+VIP客户识别与维护策略 |
| F8.6 HOS分历史趋势（v5新增） | HOS总分/基础分/减分项/奖励分的历史趋势图 |

**业务规则**：

1. 客户分级标准（按年消费额）：AAA≥5万、AA 2-5万、A 5000-2万、B 1000-5000、C<1000
2. 顾客累计数据维度：首次到店日期、最近到店日期、累计入住次数、累计间夜数、累计消费金额、常住房型、偏好渠道
3. 订单分析需区分"订单数"和"间夜量"（#12），这是两个不同指标
4. 客源结构分析按渠道（美团/携程/飞猪/直连/协议）和客类（新客/老客/会员/散客）交叉分析
5. 支持按时间段筛选分析
6. **客源结构深度分析**（v5吸纳超级店长）：增加本地/异地占比+休闲/商旅占比+价格敏感度分布；超级店长仅展示数据，睿寓智营联动定价策略——商务客多→周内稳价保收益；旅游客多→节假日提前布局涨价
7. **消费排名前20**（v5吸纳超级店长）：客人姓名+手机号+总房价/ADR+间夜数/订单数+最后联系时间；用于VIP客户识别和精准维护
8. **HOS分历史趋势**（v5吸纳超级店长）：展示HOS总分/基础分/减分项/奖励分4个维度的趋势图，帮助定位分数波动原因

**输入数据**：

| 数据项 | 数据来源 | 备注 |
|--------|----------|------|
| 订单交易数据 | PMS/OTA API | 201字段 |
| 入住历史数据 | PMS | 201字段 |
| 协议单位入住间夜数 | PMS（I-08） | 201字段 |
| 会员入住间夜数 | PMS（I-09） | 201字段 |
| 自来散客入住间夜数 | PMS（I-10） | 201字段 |
| 钟点房销售间夜数 | PMS（I-11） | 201字段 |
| 会员开卡数/储值金额 | 会员系统（I-27） | 201字段 |
| 流失订单数据 | OTA后台（O-13~O-16） | 201字段 |
| 用户来源数据（v5新增） | OTA API | 本地/异地+休闲/商旅+价格敏感度 |
| HOS分历史数据（v5新增） | OTA后台 | 总分/基础分/减分项/奖励分 |

**输出内容**：

- 订单分析报表（多维交叉分析）
- 顾客累计到店数据表（按客户维度）
- 客户分级分布图
- 客源结构分析报告（含本地/异地+休闲/商旅+价格敏感度）（v5增强）
- 高价值客户清单
- 消费排名前20客户+VIP维护策略（v5新增）
- 客源结构→定价策略联动建议（v5新增）
- HOS分历史趋势图（v5新增）

**交互方式**：手动触发分析，支持自定义时间范围和筛选条件

**竞品对标**：

| 维度 | 甩手掌柜 | 超级店长 | 睿寓智营 |
|------|----------|----------|----------|
| 客户分级 | 无客户分级功能 | ✅ 消费排名前20（含GMV/ADR/间夜数/订单数/最后联系时间） | **吸纳+增强**：消费排名+5级客户分级+VIP维护策略建议 |
| 客源结构 | 无 | ✅ 协议/会员/散客/OTA占比+本地/异地+休闲/商旅+价格敏感度 | **吸纳+联动**：深度客源分析+联动定价策略（商务客多→周内稳价；旅游客多→节假日涨价） |
| HOS趋势 | 有HOS分诊断 | ✅ HOS分历史趋势（总分/基础分/减分项/奖励分） | **吸纳**：HOS分4维度趋势图，帮助定位分数波动原因 |

### 需求9：前台协同（新增）

**需求描述**：带领前台做收益最大化，前台8项SOP数字化+指令下达+执行反馈闭环（#17）

**功能拆解**：

| 子功能 | 说明 |
|--------|------|
| F9.1 前台8项SOP数字化 | 将前台八项日常工作标准化并数字化推送 |
| F9.2 指令下达机制 | 系统自动生成控房/控价/查外网等操作指令推送给前台；v5增加微信
